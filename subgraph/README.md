# Early Repair Subgraph

Goldsky-compatible subgraph for answering where a piece can be repaired from using FWSS and PDPVerifier events.

The subgraph indexes FWSS dataset/piece context from `FilecoinWarmStorageService` and piece removals from `PDPVerifier`. Piece replicas come from FWSS `PieceAdded` events and are linked to the FWSS dataset that owns the replica. PDP `PiecesRemoved` events mark matching FWSS replicas as removed.

```bash
pnpm run deploy:mainnet
pnpm run deploy:calibration
```

Adjust the `early-repair/mainnet_0.3.0` and `early-repair/calibration_0.3.0` names in `package.json` if the Goldsky product/version should differ.

`subgraph.yaml` is intentionally generated by `prepare:mainnet` or `prepare:calibration` for Goldsky deploys. The maintained source manifests are `subgraph_mainnet.yaml` and `subgraph_calibration.yaml`.

## Repair Query

The repair model is intentionally small:

- `Piece`: one global record per CID.
- `PieceReplica`: one observed FWSS `PieceAdded` occurrence for that CID in a dataset. PDP `PiecesRemoved` events set `removed` on matching replicas.
- `DataSet`: provider and lifecycle context for the dataset that contains the replica.

`Piece.id` is the CID bytes encoded as hex, so CID lookups use Graph Node's primary-key path. Use `id_gt` cursors instead of `skip` for large scans.

To find pieces that would be lost with a provider, page over active replicas by denormalized `providerId`:

```graphql
query LostProviderPieces($providerId: BigInt!, $lastId: ID!, $first: Int!) {
  pieceReplicas(
    first: $first
    orderBy: id
    orderDirection: asc
    where: {
      id_gt: $lastId
      providerId: $providerId
      removed: false
      dataSet_: {
        initialized: true
        isActive: true
        isDeleted: false
        isEmpty: false
        pdpPaymentTerminated: false
      }
    }
  ) {
    id
    providerId
    dataSetId
    pieceId
    piece {
      id
      cid
      rawSize
    }
    dataSet {
      serviceProvider
      payee
      payer
    }
  }
}
```

Deduplicate the returned `piece.id` values client-side. Then find replacement replicas for those pieces, excluding the lost provider:

```graphql
query ReplacementReplicas($pieceIds: [ID!]!, $lostProviderId: BigInt!, $lastId: ID!, $first: Int!) {
  pieceReplicas(
    first: $first
    orderBy: id
    orderDirection: asc
    where: {
      id_gt: $lastId
      piece_in: $pieceIds
      providerId_not: $lostProviderId
      removed: false
      dataSet_: {
        initialized: true
        isActive: true
        isDeleted: false
        isEmpty: false
        pdpPaymentTerminated: false
      }
    }
  ) {
    id
    providerId
    dataSetId
    pieceId
    piece {
      id
      cid
      rawSize
    }
    dataSet {
      serviceProvider
      payee
      payer
    }
  }
}
```

`replicaCount` and `totalPieces` are historical `PieceAdded` counters. `activeReplicaCount`, `activePieces`, `PieceReplica.removed`, and `DataSet.isEmpty` account for PDP `PiecesRemoved` events observed by this subgraph. `DataSet.isDeleted` tracks PDP `DataSetDeleted` events.
