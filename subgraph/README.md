# Early Repair Subgraph

Goldsky-compatible subgraph for answering where a piece can be repaired from using FWSS events.

The subgraph indexes a single contract per deployment: `FilecoinWarmStorageService`. It does not subscribe to PDPVerifier events. Piece replicas come from FWSS `PieceAdded` events and are linked to the FWSS dataset that currently owns the replica.

pnpm install

```

For calibration:

pnpm run codegen:calibration
pnpm run build:calibration
```

## Deploy

```bash
pnpm run deploy:mainnet
pnpm run deploy:calibration
```

Adjust the `early-repair/mainnet_0.1.0` and `early-repair/calibration_0.1.0` names in `package.json` if the Goldsky product/version should differ.

`subgraph.yaml` is intentionally generated by `prepare:mainnet` or `prepare:calibration` for Goldsky deploys. The maintained source manifests are `subgraph_mainnet.yaml` and `subgraph_calibration.yaml`.

## Repair Query

The repair model is intentionally small:

- `Piece`: one global record per CID.
- `PieceReplica`: one observed FWSS `PieceAdded` occurrence for that CID in a dataset.
- `DataSet`: provider and lifecycle context for the dataset that contains the replica.

`Piece.id` is the CID bytes encoded as hex, so CID lookups use Graph Node's primary-key path. To find repair candidates for a piece across other datasets, load the `Piece` by ID and exclude the failed dataset from the derived replicas:

```graphql
query RepairCandidates($cidHex: String!, $failedDataSetId: BigInt!) {
  piece(id: $cidHex) {
    cidHex
    rawSize
    replicaCount
    replicas(where: { dataSetId_not: $failedDataSetId }) {
      pieceId
      dataSet {
        dataSetId
        providerId
        serviceProvider
        payee
        payer
        isActive
        pdpPaymentTerminated
        pdpEndEpoch
      }
    }
  }
}
```

To inspect all known replicas for a CID:

```graphql
query RepairCandidates($cidHex: ID!) {
  piece(id: $cidHex) {
    cidHex
    rawSize
    replicaCount
    replicas {
      pieceId
      dataSet {
        dataSetId
        providerId
        serviceProvider
        payee
        payer
        isActive
        pdpPaymentTerminated
        pdpEndEpoch
      }
    }
  }
}
```

Because this is intentionally FWSS-only, the subgraph does not see PDP `PiecesRemoved` events. Treat `DataSet.isActive` and payment termination fields as repair-candidate filters, then cross-check current PDP piece liveness if strict active-piece state is required.
