# Early Repair Subgraph

Goldsky-compatible subgraph for answering where a piece can be repaired from using FWSS and PDPVerifier events.

The subgraph indexes FWSS dataset/piece context from `FilecoinWarmStorageService` and piece removals from `PDPVerifier`. Piece replicas come from FWSS `PieceAdded` events and are linked to the FWSS dataset that owns the replica. PDP `PiecesRemoved` events mark matching FWSS replicas as removed.

## Deploy

```bash
pnpm run deploy:mainnet
pnpm run deploy:calibration
```

Adjust the `early-repair/mainnet_0.2.0` and `early-repair/calibration_0.2.0` names in `package.json` if the Goldsky product/version should differ.

`subgraph.yaml` is intentionally generated by `prepare:mainnet` or `prepare:calibration` for Goldsky deploys. The maintained source manifests are `subgraph_mainnet.yaml` and `subgraph_calibration.yaml`.

## Repair Query

The repair model is intentionally small:

- `Piece`: one global record per CID.
- `PieceReplica`: one observed FWSS `PieceAdded` occurrence for that CID in a dataset. PDP `PiecesRemoved` events set `removed` on matching replicas.
- `DataSet`: provider and lifecycle context for the dataset that contains the replica.

`Piece.id` is the CID bytes encoded as hex, so CID lookups use Graph Node's primary-key path. To find repair candidates for a piece across other datasets, load the `Piece` by ID and exclude the failed dataset from the derived replicas:

```graphql
query RepairCandidates($cidHex: ID!, $failedDataSetId: BigInt!) {
  piece(id: $cidHex) {
    cidHex
    rawSize
    replicaCount
    activeReplicaCount
    replicas(where: { dataSetId_not: $failedDataSetId, removed: false }) {
      pieceId
      dataSet {
        dataSetId
        providerId
        serviceProvider
        payee
        payer
        isActive
        isEmpty
        pdpPaymentTerminated
        pdpEndEpoch
      }
    }
  }
}
```

To inspect all known replicas for a CID:

```graphql
query RepairCandidates($cidHex: ID!) {
  piece(id: $cidHex) {
    cidHex
    rawSize
    replicaCount
    activeReplicaCount
    replicas {
      pieceId
      removed
      dataSet {
        dataSetId
        providerId
        serviceProvider
        payee
        payer
        isActive
        isEmpty
        pdpPaymentTerminated
        pdpEndEpoch
      }
    }
  }
}
```

`replicaCount` and `totalPieces` are historical `PieceAdded` counters. `activeReplicaCount`, `activePieces`, `PieceReplica.removed`, and `DataSet.isEmpty` account for PDP `PiecesRemoved` and `DataSetEmpty` events observed by this subgraph.
